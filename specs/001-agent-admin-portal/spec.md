# Feature Specification: 代理商管理後台系統

**Feature Branch**: `001-agent-admin-portal`  
**Created**: 2026-04-21  
**Status**: Draft  
**Input**: User description: "代理商管理後台系統，使用 Element Plus Admin 框架，包含登入功能與代理商列表頁面"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 管理員登入 (Priority: P1)

管理員在瀏覽器開啟後台，看到登入頁面，輸入帳號密碼後成功進入後台首頁。若帳密錯誤，系統顯示錯誤提示並保持在登入頁。

**Why this priority**: 所有後台功能皆需登入後才可存取，登入是整個系統的入口，必須最先完成。

**Independent Test**: 開啟登入頁面，輸入正確帳號密碼後被導向後台主頁，即視為可測試的 MVP。

**Acceptance Scenarios**:

1. **Given** 使用者未登入並訪問登入頁，**When** 輸入正確的帳號與密碼並點擊登入，**Then** 系統驗證成功並將使用者導向後台主頁。
2. **Given** 使用者在登入頁，**When** 輸入錯誤的帳號或密碼，**Then** 系統顯示「帳號或密碼錯誤」的提示訊息，並保持在登入頁。
3. **Given** 使用者已登入，**When** 直接訪問登入頁，**Then** 系統自動導向後台主頁，不再顯示登入頁。
4. **Given** 使用者已登入，**When** 點擊登出，**Then** 清除登入狀態並跳回登入頁。
5. **Given** 使用者未登入，**When** 直接訪問任意後台功能頁面，**Then** 系統自動導向登入頁。

---

### User Story 2 - 代理商列表瀏覽 (Priority: P2)

登入後的管理員可以在代理商列表頁中查看所有代理商資料，支援分頁顯示，並可依條件搜尋。

**Why this priority**: 代理商列表是系統的核心業務頁面，是後續所有代理商管理功能（新增、編輯、停用）的基礎。

**Independent Test**: 登入後訪問代理商列表頁，可看到代理商資料表格（含分頁），即視為可獨立測試的功能切片。

**Acceptance Scenarios**:

1. **Given** 管理員已登入，**When** 訪問代理商列表頁，**Then** 頁面顯示代理商資料表格，欄位包含：代理商 ID、代理商名稱、聯絡人、聯絡電話、狀態、建立時間、操作（狀態切換：停用/啟用）。
2. **Given** 代理商列表頁，**When** 系統載入資料，**Then** 顯示資料表格，預設依建立時間降冪排列，每頁預設顯示 20 筆。
3. **Given** 代理商列表頁，**When** 管理員點擊任意欄位的欄標題，**Then** 表格依該欄位切換升冪/降冪排列，並重置至第一頁。
4. **Given** 代理商列表頁，**When** 管理員在搜尋列輸入代理商名稱並點擊搜尋，**Then** 表格僅顯示名稱符合關鍵字的代理商，分頁重置至第一頁。
5. **Given** 代理商列表頁，**When** 管理員從狀態篩選下拉選單選擇「啟用」或「停用」，**Then** 表格僅顯示符合所選狀態的代理商，分頁重置至第一頁。
6. **Given** 代理商列表頁，**When** 管理員同時設定名稱關鍵字與狀態篩選並點擊搜尋，**Then** 表格顯示同時符合名稱與狀態條件的代理商。
7. **Given** 代理商列表有多頁資料，**When** 管理員切換頁碼，**Then** 表格載入對應頁的資料，並顯示總筆數。
8. **Given** 代理商列表資料載入中，**When** 資料請求尚未完成，**Then** 表格顯示載入狀態，避免使用者誤以為無資料。
9. **Given** 管理員在代理商列表頁，**When** 點擊某代理商「操作」欄的狀態切換按鈕，**Then** 系統送出狀態更新請求，成功後表格中該代理商的狀態欄即時更新，切換按鈕文字對應變更。
10. **Given** 管理員點擊狀態切換按鈕，**When** 後端回應失敗，**Then** 系統顯示錯誤提示訊息，代理商狀態維持不變。

---

### Edge Cases

- 當後端服務請求失敗或逾時時，系統顯示錯誤提示訊息，而非白頁或靜默失敗。
- 當代理商列表無資料時，頁面顯示空狀態提示文字「暫無資料」。
- 當登入憑證過期時（後端回應 401），系統立即清除登入狀態並導向登入頁，不嘗試自動刷新 Token。
- 當使用者在搜尋欄輸入空字串並點擊搜尋時，系統重置並載入完整列表。
- 登入表單送出期間防止重複提交，按鈕進入不可點擊狀態直至請求完成。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 提供登入頁面，包含帳號與密碼輸入欄位及登入按鈕。
- **FR-002**: 系統 MUST 在登入成功後儲存使用者憑證，並導向後台主頁。
- **FR-003**: 系統 MUST 對所有需要驗證的頁面進行存取控制，未登入者導向登入頁。
- **FR-004**: 系統 MUST 提供代理商列表頁，以表格形式展示代理商資料。
- **FR-005**: 代理商列表 MUST 支援分頁功能，透過後端分頁載入資料。
- **FR-006**: 代理商列表 MUST 支援依代理商名稱進行關鍵字搜尋（後端過濾）。
- **FR-006a**: 代理商列表 MUST 支援依狀態（全部 / 啟用 / 停用）進行篩選，可與名稱關鍵字搜尋同時使用（後端複合過濾）。
- **FR-006b**: 代理商列表 MUST 支援所有欄位的升冪/降冪排序，預設依建立時間降冪排列；欄位排序透過後端排序參數實現，切換排序時重置至第一頁。
- **FR-007**: 系統 MUST 在資料請求期間顯示載入狀態，在失敗時顯示對應錯誤提示。
- **FR-008**: 系統 MUST 提供側邊欄導覽選單，包含「代理商管理 > 代理商列表」選項。
- **FR-009**: 系統 MUST 在頂部工具列提供登出功能，點擊後清除登入狀態並導向登入頁。
- **FR-010**: 系統 MUST 在收到後端 401 回應時立即清除登入憑證並將使用者導向登入頁；不實作 Token 自動刷新機制。
- **FR-011**: 代理商列表的「操作」欄 MUST 提供狀態切換按鈕，管理員可直接將代理商狀態在「啟用（active）」與「停用（inactive）」之間切換，切換後表格立即反映最新狀態。

### Key Entities

- **管理員 (Admin)**: 後台操作使用者，擁有帳號與密碼，登入後取得存取憑證；屬性：`id`、`username`、`email`、`token`。
- **代理商 (Agent)**: 系統管理的核心業務實體；屬性：`id`（唯一識別碼）、`name`（代理商名稱）、`contactPerson`（聯絡人）、`contactPhone`（聯絡電話）、`status`（`active` / `inactive`）、`createdAt`（建立時間）。
- **分頁回應 (Paginated Response)**: 列表資料回應結構；屬性：`data`（列表陣列）、`total`（總筆數）、`page`（當前頁）、`pageSize`（每頁筆數）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 管理員可在 30 秒內完成登入並進入代理商列表頁。
- **SC-002**: 代理商列表頁於資料回應後 2 秒內完成渲染並可供操作。
- **SC-003**: 搜尋功能可在使用者觸發後 1 秒內（排除網路延遲）更新表格結果。
- **SC-004**: 所有後端錯誤（包含請求失敗與網路斷線）皆有對應的使用者可見提示，不出現無法操作的空白狀態。
- **SC-005**: 未登入的使用者無法直接存取任何後台頁面，所有受保護頁面均執行存取控制。

## Assumptions

- 後端 API 已存在或將同步開發，前端依照預定 API 契約對接（登入端點、代理商列表端點）。
- 本階段（v1）僅實作「登入」、「代理商列表」及「代理商狀態切換（停用/啟用）」三個功能，不包含新增、編輯、刪除代理商。
- 使用預設的後台佈局（側邊欄 + 頂部導覽列），不進行深度客製化。
- 僅支援桌面瀏覽器（Chrome、Edge 最新版），不要求響應式設計。
- 代理商資料量預估在 10,000 筆以內，分頁足以處理效能需求。
- 登入採用帳號密碼驗證，不涉及第三方登入、單一登入（SSO）或雙因素認證（2FA）。
- 登入憑證（Token）有效期為 8 小時；到期後使用者需重新登入，系統不提供自動刷新。

## Clarifications

### Session 2026-04-21

- Q: v1 代理商列表「操作」欄應顯示哪些操作？ → A: 僅顯示狀態切換按鈕（停用/啟用），不包含編輯或刪除。
- Q: Token 過期時應採用靜默刷新還是強制重新登入？ → A: 強制重新登入——後端回應 401 時立即清除 Token 並導向登入頁，不實作 Refresh Token 機制。
- Q: 預期的 Token 有效期（Session 時長）為何？ → A: 8 小時（一個工作日），到期後強制重新登入。
- Q: 代理商列表是否需要支援欄位排序？ → A: 所有欄位均可排序（升冪/降冪），後端排序，切換時重置分頁。
- Q: 代理商列表是否需要狀態篩選功能？ → A: 是，新增狀態下拉篩選（全部 / 啟用 / 停用），可與名稱搜尋複合使用。
